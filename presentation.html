🧠 How the Pre-Screener Works

1. Feature Engineering

def extract_features(login_event): return { 'country': 'NO', # Geographic location 'device': 'tablet', # Device type 'ua_family': 'chrome', # Browser family 'hour': 14, # Time of day 'weekday': 2, # Day of week 'rtt_ms': 641.0, # Network latency 'login_success': True # Authentication result }

2. Anomaly Scoring Process

  • Input: 7-dimensional feature vector
  • Processing: 200 decision trees (Isolation Forest)
  • Output: Continuous anomaly score (-0.14 to +0.08)
  • Decision: Score ≥ threshold → Send to LLM agents

🎯 Why Isolation Forest?

Algorithm Comparison

Requirement Isolation Forest Neural Networks SVM K-Means
Speed (<1ms) ✅ Fast ❌ 10-100ms ❌ Slow ✅ Fast
Unsupervised ✅ No labels ❌ Needs labels ❌ Needs labels ✅ No labels
Mixed Features ✅ Categorical + Numerical ✅ With preprocessing ❌ Preprocessing heavy ❌ Numerical only
Explainable ✅ Tree decisions ❌ Black box ❌ Complex ✅ Distance-based
Memory ✅ 1MB model ❌ GBs required ❌ Stores vectors ✅ Small
